What gets entered: your browsing history from Chrome and Safari, watermarked and synced on a schedule, ninety days of backfill on first run. Apple Notes, read in place. Files and PDFs you hand it with shyn ingest. And anything Claude decides is worth keeping, via the remember tool.

The screen, read gently

A capture agent OCRs what's on screen as you work — deduplicated, throttled, excludable per-app (shyn exclude), pausable from the menu bar, and swept by a retention window. The PDF you skimmed once is in the register even if you never saved it.

Meetings, kept whole

Microphone and system audio, transcribed locally — no bot joins your call, nothing is uploaded. A pre-roll gate means phantom meetings don't get invented; sustained silence means the entry closes itself.

Search that works from second zero

Keyword search works the moment the daemon starts; the embedding model downloads in the background and semantic search switches on when it's ready. Every result says which mode answered you.

Safari history and Apple Notes need Full Disk Access — shyn says so plainly in the popover instead of silently reading nothing.

shyn diagnosewhen something's off: a copy-pasteable, content-free support block.

The menu bar is the manual

Click the ☀️: service health, index size, this week's numbers, pause buttons, a live meeting banner with Stop / Cancel, and a first-run checklist that walks every permission with deep links into System Settings.

Connect your AI — any of them

shyn speaks MCP, the open protocol. Anything that runs local MCP servers connects:
Claude Code — claude mcp add shyn (printed by setup)
Claude Desktop — drag in shyn.mcpb
Cursor / Windsurf — add to ~/.cursor/mcp.json:
{ "mcpServers": { "shyn": { "command":
  "~/Library/Application Support/shyn/bin/shyn-mcp" } } }
VS Code (agent mode), Zed, Gemini CLI, Raycast — same command, their config file.
Five tools appear either way: search_memory, recent_activity, remember, forget, memory_status.
ChatGPT is the one exception — it only accepts remote MCP servers, and shyn will never tunnel your memory through someone's cloud.

What shyn never does.

    
Not a policy page — an architecture. There is no server to call. Grep the code.

    

      
never
Phones home
No telemetry, no analytics, no crash reporter. There is nowhere for your data to go.

      
never
Stores your searches
Stats are counts in your local database. The number of searches — never the words.

      
never
Ships your content
When something breaks, shyn diagnose prints a content-free block. You choose where it goes.

    


    

      
Objections, answered

      
Isn't this surveillance?Of yourself, by yourself, for yourself — on hardware you own, with a forget command and a pause button in the menu bar.

      
Where does my data live?One encrypted SQLite file under ~/Library/Application Support/shyn. The key lives in your macOS Keychain. Delete the folder and it's gone.

      
Why not a cloud notetaker?Because your days shouldn't be a vendor's dataset. No bot in your meetings, no upload, no account — and it works on a plane.

      
What does it cost to run?A background daemon and a ~640MB local model, downloaded once. Search stays under half a second at 100k chunks — on your laptop.

      
What breaks?It's pre-alpha and says so: known issues ship in the repo, honestly. shyn diagnose exists because friends' machines break in new ways.

      
Can I really uninstall it?shyn uninstall --purge removes services, database, logs, and the Keychain key. Then brew uninstall shyn. Nothing lingers.

Homebrew checks the sha256. Apple Silicon, pre-alpha, honestly self-signed.

      
ii.
Set up
shyn setup starts everything and walks you through permissions — the ☀️ shows a live checklist. If macOS doesn't list an app, add it from ~/Library/Application Support/shyn/bin.

      
iii.
Connect your AI
Claude, Cursor, VS Code, Zed, Gemini CLI — one config line each (shyn speaks MCP, the open protocol). From then on, your AI just remembers.

    


    
Fine print, honestly: shyn is pre-alpha. The apps are self-signed and re-signed locally by shyn setup, which also clears the download quarantine with your consent — Apple-notarized one-click installs are on the roadmap. Screen and audio capture need permissions macOS will ask you for; shyn's popover explains each one before you grant it.
